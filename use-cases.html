Boundary

Restart

A project resumes after a pause without relying only on chat memory or guesses about what mattered last time. Important status, wording, risks, and next actions remain recoverable.

Review

A project owner checks AI output before treating it as a project fact, approved decision, or permission to act. Candidate issues can stay visible while the evidence is still being reviewed.

Handoff

Another person or future session can see what changed, what is approved, what remains open, and what should happen next instead of reconstructing the project from fragments.

Continuity

The project carries its working context forward through durable records rather than depending only on chat memory.

Approval

Consequential actions remain separate from simple suggestions, and permission stays explicit.

Evidence

Claims stay tied to source status, review scope, and visible limits instead of being treated as true by momentum alone.
