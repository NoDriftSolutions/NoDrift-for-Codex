NoDrift Does Not Control AI

In human communication, we do not control the transmission. We control the reception: what we accept, verify, record, and act on.

NoDrift applies that same discipline to AI-assisted project work.

  • It does not change the LLM or guarantee every answer is true.
  • It does not add NoDrift files to outgoing prompts or conversation turns.
  • It checks important claims before they become project facts.
  • It keeps decisions, open questions, and source boundaries visible.
  • It pauses unclear actions until approval is clear.
  • It turns recognized errors into private correction guidance for future work.

NoDrift FAQ

Short answers for setup, operation, memory, customization, evidence, payment, and platform support.

Start Here

What is NoDrift?

NoDrift is a buyer-only governed workspace system for Codex. It gives serious project work source checks, memory support, approval boundaries, correction habits, and evidence records.

Is this software or a separate app?

No. NoDrift is not a dashboard, plugin, executable, or background monitor. It works through buyer-only workspace materials used inside a Codex project workspace.

Is this just a prompt pack?

No. Prompts are included, but the product is the protected workspace structure. The exact file list and operating language are private internal information.

How do I start?

After purchase, the buyer setup guide walks you through installation and first use. The public site does not publish the exact setup sequence.

Do I fill out the files by hand?

NoDrift is not meant to become paperwork. Codex helps maintain the records during work; the user answers questions, approves important actions, and questions anything that does not sound right.

How long does setup take?

The first setup pass is designed to be light. Deeper customization and verification can happen later inside the buyer workspace.

How It Works

Does NoDrift control the LLM?

No. The LLM generates the response. NoDrift governs reception: what the work session accepts, verifies, records, corrects, and acts on after output arrives.

Does it attach files to outgoing messages?

No. NoDrift does not attach its buyer-only workspace materials to outgoing messages or conversation turns.

What happens in a normal session?

The user explains the work. Codex works inside the project workspace. NoDrift keeps sources, decisions, risks, continuity, and approval boundaries visible.

What if something sounds wrong?

Question it directly and politely. NoDrift can treat the issue as a candidate error, preserve the relevant exchange, classify the likely problem, and prepare a correction path.

What does Critical mean?

Critical damage means an error may affect files, publishing, payments, external actions, or serious project direction. Important but non-damaging errors can still become private correction guidance.

Does NoDrift guarantee perfect answers?

No. NoDrift reduces risk, preserves corrections, and makes unsupported claims easier to catch. It does not guarantee that every mistake will be prevented or detected.

Customization, Memory, And Updates

Can I customize how Codex talks to me?

Yes. The working profile can set communication style, detail level, progress updates, teaching mode, uncertainty reporting, question style, token-use preferences, and whether Codex leads by recommending the next safe directive.

Does customization weaken the rules?

No. Customization changes communication and workflow style. It does not override source discipline, privacy, approval gates, verification, evidence requirements for broad claims, or the rule against inventing facts.

What affects token use?

Communication detail has the largest effect. NoDrift files are not attached to outgoing messages; token use changes mainly when the user asks for more explanation, status detail, teaching, or evidence labels.

What are project memory files?

They preserve current state, source boundaries, open decisions, risks, handoffs, and action status outside fragile chat history.

What are private continuity records?

They are buyer-only navigation records for serious work. The exact format and operating rules are private internal information.

How do private correction updates work?

Reusable correction patterns can be delivered as buyer-safe updates. The exact update format is private internal information.

What if something goes wrong?

If you are already working in your NoDrift workspace, type troubleshooting in Codex. The Codex agent should pause forward work, check only what is needed, explain what it found, and ask for go before changing anything.
