Quickstart

Quickstart explains startup in brief: place the governance package in the project folder, use begin, let the AI Orchestrator read and activate governance, then describe the project and start work.

1. Place The Governance Package In The Project Folder

NoDrift starts inside the actual project folder. The governance package lives there so the active AI project app can work with the same project, records, boundaries, and controls from the first moment of setup.

2. Open The Project In The AI Orchestrator And Type begin

The startup trigger is simple. Open the project as the active AI Orchestrator workspace and use begin to start the NoDrift startup routine.

3. The AI Orchestrator Reads The Governance Files

The AI Orchestrator reads the root AGENTS.md and the installed governance package, then loads the project boundaries, startup rules, approval logic, source discipline, and active records needed for governed work.

4. The AI Orchestrator Configures Governance, Then Asks For The Project

Once startup is complete, the AI Orchestrator asks the user to describe the project. Work then begins under active governance, with source, approval, continuity, evidence, and correction controls already in place.

5. Deepen Later

The first pass is meant to get the workspace active quickly. Deeper customization, stronger verification routines, and extended governance options can be added later without blocking startup.
