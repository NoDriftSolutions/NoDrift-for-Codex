Quickstart

Quickstart separates two steps: portfolio installation places NoDrift in the project folder, and governed startup begins after the project is opened and the user types begin.

1. Install The Portfolio In The Project Folder

NoDrift starts with the actual project folder. Portfolio installation places the protected workspace materials there so the active AI project app can work with the same project, records, boundaries, and controls.

2. Open The Project And Type begin

begin does not install the portfolio. It starts governed startup after the project has been opened as the active AI Orchestrator workspace.

3. The AI Orchestrator Reads The Installed Governance

The AI Orchestrator reads the root AGENTS.md and the installed governance files, then loads the project boundaries, startup rules, approval logic, source discipline, and active records needed for governed work.

4. Governed Startup Asks For The Project

Once governed startup is active, the AI Orchestrator asks the user to describe the project. Work then begins under source, approval, continuity, evidence, and correction controls already in place.

5. Deepen Later

The first pass is meant to get the workspace active quickly. Deeper customization, stronger verification routines, and extended governance options can be added later without blocking startup.
