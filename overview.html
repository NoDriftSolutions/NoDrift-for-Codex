Workflow

NoDrift keeps AI work moving in order: start clear, check the answer, approve what matters, and continue without starting over.

NoDrift Continuity PathOne project thread, carried forward without starting over.

NoDrift keeps each AI work session tied to source truth, approval, heartbeats, corrections, branches, evidence, and carryover.

  1. Start Clear

    Name the real task, current goal, and what matters before the AI starts moving.

  2. Set Boundary

    Keep today's work from quietly expanding into side quests or future features.

  3. Use Source Truth

    Use project files and records as proof. Chat memory is not the authority.

  4. Run Heartbeats

    Use programmed refreshes and behavioral checks to route risk back into governance.

  5. Check Claim

    Separate confident AI output from verified fact before relying on it.

  6. Ask Approval

    Important moves wait for the user's exact yes before the work changes.

  7. Make Change

    Do the approved work without widening the task or changing direction.

  8. Verify Result

    Check what actually changed, not just what the AI says changed.

  9. Correct Drift

    Name wrong assumptions and preserve the fix so the same error stops returning.

  10. Branch When Needed

    Give separate objectives, risks, or decision paths their own record.

  11. Record Evidence

    Keep proof, limits, and open questions connected to the claim.

  12. Carry Forward

    Let the next session resume from the record instead of restarting.

12 Programmed Heartbeats

NoDrift maps trigger points like startup, phase change, readiness, public/private boundaries, external action, package work, payment, compaction, source conflict, correction, token burn-rate, and website claims to the right governance refresh.

6 Behavioral Heartbeats

Behavioral Heartbeats respond when the user calls out drift, over-effort, wordiness, stubbornness, unsupported source claims, or governance-operation problems. The assistant may also initiate the same check as a self-audit.

Correction Protocol

Heartbeats do not control the model or grant permission. They route the session back to correction, source truth, approval boundaries, and the next safe action.
