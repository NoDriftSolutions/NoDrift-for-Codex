Approval gates

Why Go Is Not A Blank Check

A visible "go" should approve the next bounded task, not every adjacent idea the AI agent can imagine.

NoDrift conceptGo boundary
User problemThe AI agent expands approval beyond the named action.
Reading depthFoundation

The problem

In fast work, "go" can be misread as broad permission. The AI agent may complete the approved task, then continue into nearby changes that were never approved.

The rule

Approval should attach to a specific object and a specific action. If the next action is different, it needs its own approval.

How NoDrift helps

NoDrift keeps approval literal. A narrow approval stays narrow even when the AI agent sees other tempting work.

Practical hint

Before saying "go," name the action: "Go: update only the blog index labels." That protects the project from helpful overreach.
