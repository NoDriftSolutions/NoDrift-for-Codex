Approval gates

Why External Actions Need A Harder Gate

Local work can usually be reviewed and repaired. External action can transmit, publish, spend, or expose.

NoDrift conceptExternal action gate
User problemThe AI agent treats external action like ordinary local work.
Reading depthAdvanced

The problem

Not all actions have the same risk. Editing a local draft is different from publishing, uploading, paying, sending, sharing, or changing access.

The harder gate

External action should require exact approval at the moment of action. The user should know what will happen, where it will happen, and what information or authority is involved.

How NoDrift helps

NoDrift makes external-action boundaries visible so the AI agent does not treat access as permission.

Practical hint

Use direct language: "No external action is approved." When external action is needed, approve that exact action separately.
