NoDrift Family of Products

NoDrift is a family of AI project-control products for builders, programmers, freelancers, teams, managers, and organizations working across different AI apps. Each product path adapts NoDrift to the tool the user actually touches while keeping the same core purpose: stronger source truth, clearer approvals, better continuity, organized corrections, evidence-backed decisions, safer handoffs, and more reliable readiness review. The free Demonstrator shows the working behavior, Basic gives a single user the first serious package, Extended adds deeper controls for longer professional work, and Teams / Enterprise brings NoDrift into shared projects where owners, managers, programmers, and reviewers need the same record.

Choose App Codex

NoDrift for Codex supports the way a user actually works in Codex: opening a project, asking Codex to inspect files, using AGENTS.md for persistent instructions, starting with /init, checking changes with /review or /diff, and controlling action through /permissions, sandbox, and approval settings. NoDrift keeps those touchpoints organized so Codex does not drift from source truth, approval boundaries, or the current project state while the user moves from question to edit to handoff. Extended

Price pending

Everything in
Basic, plus:

  1. Programmed governance Heartbeats. Adds 12 trigger-based governance refreshes for startup, phase change, readiness, public/private boundaries, external action, package or release work, payment or delivery, compaction, source conflict, correction, token burn-rate, and website or campaign claims.
  2. Behavioral Heartbeats. Adds six named behavior checks for drift, truth/source, effort, brevity, stubbornness, and governance operation. The user can trigger them, and the assistant can run them as a self-audit when a risk pattern appears.
  3. Heartbeat-to-correction routing. Routes user-reported or self-audited behavior problems into correction protocol so the issue is named, the approved boundary is restored, and the correction is preserved when required.
  4. Carryover Capsules. Gives the user a fuller handoff between chats or sessions, including current goal, decisions, boundaries, evidence, blockers, and next safe action.
  5. Governance Check Prompts. Lets the user ask the AI to check whether it is still connected to the current NoDrift rules, project files, approval boundaries, and working agreements.
  6. Readiness Review Gates. Adds a stronger review before anything is called complete, tested, buyer-ready, publish-ready, delivery-ready, or safe to rely on.
  7. Evidence Tracking. Connects claims, decisions, and status updates to the files, tests, approvals, or records that support them.
  8. Decision And Risk Tracking. Records important decisions, open risks, unresolved questions, and items that should not move forward without approval.
  9. Control Dashboard Support. Gives the user a clearer view of active work, blocked work, pending approvals, evidence gaps, open risks, and next safe actions.
  10. Advanced Correction Carryover. Carries corrections forward so the AI does not keep repeating the same mistake after the user has already fixed it.
  11. Package And Product Readiness Support. Helps projects that involve product tiers, buyer-facing claims, website copy, test evidence, release decisions, or package readiness.
  12. Professional Continuity For Serious Solo Work. Turns NoDrift from a working individual package into a deeper professional control system for longer, higher-risk, or buyer-facing projects.

Extended is the professional single-user level for longer client, product, or package work that needs deeper continuity and readiness control.

Teams / Enterprise

Custom

Everything in
Extended, plus:

  1. Role-aware approvals. Different people can have different approval authority, so a teammate, manager, owner, client, or executive is not treated as having the same decision power.
  2. Team routing for the right decision-maker. The AI is guided to route questions, blockers, approvals, and risks to the correct person or role instead of assuming approval.
  3. Shared project memory for more than one person. Decisions, constraints, corrections, and status can carry across multiple users, not just one individual working alone.
  4. Manager and owner visibility. Senior stakeholders can see what is active, blocked, evidence-backed, waiting for approval, or not ready to move forward.
  5. Team correction and incident records. Mistakes, misunderstandings, bad AI assumptions, and repeated problems can be recorded so the whole team can learn from them.
  6. Escalation paths for higher-risk work. The AI can be guided to stop and escalate when work involves public claims, legal/security concerns, buyer-facing material, or delivery promises.
  7. Shared evidence for team decisions. Claims, approvals, readiness states, and delivery decisions can be tied to supporting files, tests, records, or approvals the team can review.
  8. Safer delegation between people and AI sessions. Work can move from one person, chat, or AI session to another with clearer context, boundaries, decisions, and next actions.
  9. Organization-level public/private boundaries. Teams can better protect private notes, internal decisions, local paths, client material, buyer-only content, and restricted information.
  10. Governance for serious multi-person AI work. Teams / Enterprise turns NoDrift into a shared operating layer for organizations that need controlled AI collaboration.

Teams / Enterprise is scoped by the number of users, approval roles, supported apps, training needs, and private setup requirements.
