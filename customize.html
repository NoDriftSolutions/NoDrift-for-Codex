Communication

The working style can be shorter, more direct, more explanatory, or more visibly labeled depending on the project owner's needs. Communication detail is also the setting with the largest effect on token use.

Workflow

Working profile, work leadership, question handling, progress updates, explanation level, language level, and evidence status can be tuned for different kinds of projects.

Always Protected

Customization does not override source discipline, privacy, approval gates, verification, evidence requirements, or the rule against inventing facts. Token savings should come from communication choices, not weaker safeguards.

Style

Communication can become shorter, more directed, more explanatory, or more technically specific.

Visibility

Progress and evidence can be surfaced in a way the project owner can actually review instead of guessing what the work session is doing.

Safeguards

The protective boundaries remain active while the working style changes, and only one working profile is active at a time.
